SEDI_SPI_IOCTL_TI_SSI 

Texas Instruments Frame Format No need to input args

SEDI_SPI_IOCTL_MICROWIRE 

National Microwire Frame Format Need to input microwire configure pointer

SEDI_SPI_IOCTL_DATA_WIDTH 

Set the number of bits per SPI frame The parameter range is [1,16] For 4 bits operation, users need to provide an expand buffer, which covert 4-bits data to 8-bits data. The transfer size shall also use the expanded buffer size while do transfer.

SEDI_SPI_IOCTL_MSB 

MSB first send out, default

SEDI_SPI_IOCTL_LSB 

LSB first send out, it should be supported by hardware

SEDI_SPI_IOCTL_CS_UNUSED 

The SPI will not drive or monitor the CS pin, user drives it with GPIO API before and after transaction, or directly connect the SPI slave's CS pin to a fixed low level.

SEDI_SPI_IOCTL_CS_HW 

The SPI auto active the CS pin before transaction, and deactive it after complete or abort. The parameter is bitmask of CS pin been controlled by SPI

SEDI_SPI_IOCTL_SPEED_SET 

Get SPI speed in Hz

SEDI_SPI_IOCTL_SPEED_GET 

Set SPI speed in Hz

SEDI_SPI_IOCTL_TX_WATERMARK_SET 

Set the FIFO depth, set it large for bulk transfer, set it small for few data transfer (register access). For bulk transfer, the data length should be aligned with the FIFO depth.

SEDI_SPI_IOCTL_RX_WATERMARK_SET 

Set the FIFO depth, set it large for bulk transfer, set it small for few data transfer (register access). For bulk transfer, the data length should be aligned with the FIFO depth.

SEDI_SPI_IOCTL_TX_WATERMARK_GET 

Get the FIFO depth

SEDI_SPI_IOCTL_RX_WATERMARK_GET 

Get the FIFO depth

SEDI_SPI_IOCTL_DUMMY_DATA 

Set the dummy data on the bus, for example, master wants to read from slave. It can be 0x00 or 0xFF, which means keep data line low level or high level.

SEDI_SPI_IOCTL_ABORT 

Abort the data transfer.

SEDI_SPI_IOCTL_LOOPBACK 

Loopback mode for test

SEDI_SPI_IOCTL_BUFFER_SETS 

SPI transfer buffer sets, it means in a transfer, the buffer can be not physical continuous. Users need to update buffer while callback event is SEDI_SPI_EVENT_TX_FINISHED or SEDI_SPI_EVENT_RX_FINISHED. Users need to use function sedi_spi_update_tx_buf or sedi_spi_update_rx_buf to update buffers. Input 1 means enable this mode, 0 means disable this mode. Notice: This mode only supported in irq transfer mode.

SEDI_SPI_IOCTL_LINE_MODE 

Set SPI line mode, it can be single/dual/quad/octal mode.

SEDI_SPI_IOCTL_NUM 

Indicates the end of ioctl code
